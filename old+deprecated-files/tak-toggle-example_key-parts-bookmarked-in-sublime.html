Introduction to CSS

CSS: an overview

About CSS

  • CSS (Cascading Style Sheets) is a language used to describe the appearance and formatting of your HTML.
  • A style sheet is a file that describes how an HTML file should look. That's it!
  • We say these style sheets are cascading because the sheets can apply formatting when more than one style applies.

There are two main reasons for separating your form/formatting (CSS) from your functional content/structure (HTML):

  1. You can apply the same formatting to several HTML elements without rewriting code (e.g. style="color:red":) over and over
  2. You can apply similar appearance and formatting to several HTML pages from a single CSS file

How to call CSS file from index.html

1 <head>
2     <link type="text/css" rel="stylesheet" href="stylesheet.css"/>
3 </head>

Syntax of CSS

The most general form

1 selector {
2     property: value; /* don't forget the semi-colon at the end */
3 }

Some definitions

  • A selector can be any HTML element, such as <p>, <img>, or <table>. You just take off the <>s!
  • A property is an aspect of a selector. For instance, you can change the font-family, color, and font-size of the text on your web pages.
  • A value is a possible setting for a property. color can be red, blue, black, or almost any color; font-family can be a whole bunch of different fonts; and so on.

Few remarks

  • You can set multiple properties at once
  • Unlike when inline styling, you don't need to surround property:value pair with quotes "
  • values are case-sensitive!!!
1 p {
2     font-family: Arial;
3     color: blue;
4     font-size: 24px;
5 }

Basic css-properties (to be aware of)

Basic property values: px vs em

  • font-size: 12px -- px is "pixels", an absolute unit
  • font-size: 12 em (not <em> tag)
    • em above is a relative measure (1em = default font-size on whatever screen the user is using)
    • great for smartphone screens!

CSS Classes and IDs

More flexibility with CSS selectors

Multiple selectors for nested HTML elements

How to select nested HTML element like the following?

1 <div>
2     <div>
3         <p>I like tacos!</p>

You can select them in css like this:

1 div div p {
2     /*CSS stuff!*/
3 }

This will **only select the <p> element nested inside 2 <div>s

WARNING (importance of commas in selectors!!)

1 .red div p {
2     /* this will precisely select the <p> that is a child of a <div> who is a child of a class "red"*/
3 }
4 
5 .red, div, p {
6     /* this will apply changes to all class "red", all <div>s, and all <p>s */
7 }

Select all with * (useful for visualizing page layout)

  • Apply CSS styling to every element on the page
  • Adding border around every element on the HTML page is useful to get an idea of the page-layout
1 *{
2     border:1px dashed #3A5FCD;
3 }

HTML document is a tree (html elements "branches out")

The HTML Tree

  • <html></html> -- the tree trunk
    • <head></head>, <body></body> -- the two big branches from the trunk
      • <h1><p><table>....whatever!!! -- another layers of branching...and so on

Parents, children, and siblings

Link -- a great exercise

Here's how the selector work:

1 parent child grand-child grand-grand-child ... {
2     /* change property narrowed to the grand-grand-child */
3 }

Grab the immediate children using >

This selector will grab any <p> that is nested somewhere inside a <div> that is nested somewhere inside a <div> that is nested somewhere inside another <div>

1 div div p { /* Some CSS */ }
  • The following only grabs <p>s that are nested directly inside of <div> s.
  • It won't grab any <p> that are, say nested inside lists that are in turn nested inside <div>s.
1 div > p {/* some css */}

Selector with higher specificity will over-ride the other selector (outside of class and IDs)

Class vs. IDs

  • Classes -- useful when you have a bunch of elements that should all receive the same styling.
    • allows you to take elements of different types and give them the same styling.
  • IDs -- great for when you have exactly one element that should receive a certain kind of styling.
    • allows you to apply style to a single instance of a selector, rather than all instances.

A pseudo-class selector is a way of accessing HTML items that aren't part of the document tree

  • For instance, it's very easy to see where a link is in the tree.
  • But where would you find information about whether a link had been clicked on or not? It isn't there!

Pseudo-class selector syntax :

1 selector:pseudo-class_selector {
2     property: value;
3 }

Pseudo-class selectors for links (eg: <a>)

  • a:link --- An unvisited link.
  • a:visited --- A visited link.
  • a:hover --- A link you're hovering your mouse over.

First-child (ELEM:first-child), n-th child (ELEM:nth-child(2))

2 p:first-child{font-family:cursive}
3 p:nth-child(2){font-family:Tahoma}
4 p:nth-child(3){color:#CC0000}
5 p:nth-child(4){color:#00FF00}

CSS Element positioning

The CSS Box Model

  • Each HTML element is like a tiny box or container that holds the pictures and text you specify.

Run this to see a layout of your html dom

*{ /* recall the * selects all html elements */
    border: 1px dashed #0000FF;
 }

The display-style and four values (block, inline-block, inline, none)

  1. block
    • This makes the element a block box.
    • It won't let anything sit next to it on the page!
    • It takes up the full width.
  2. inline-block
    • This makes the element a block box
    • Allows other elements to sit next to it on the same line.
  3. inline
    • This makes the element sit on the same line as another element, but without formatting it like a block.
    • It only takes up as much width as it needs (not the whole line).
    • better suited for HTML elements that are blocks by default, such as headers and paragraphs.
  4. none
    • This makes the element and its content disappear from the page entirely!
1 div {
2     height: 50px;
3     width: 100px;
4     border: 2px solid black;
5     border-radius: 5px;
6     display:inline-block;
7 }

CSS-box model: margin, border, padding, content styles

Try this: http://www.w3schools.com/css/tryit.asp?filename=trycss_boxmodel

  • Content - The content of the box, where text and images appear
  • Padding - Clears an area around the content. The padding is transparent
  • Border - A border that goes around the padding and content
  • Margin - Clears an area outside the border. The margin is transparent

Important: When you set the width and height properties of an element with CSS, you just set the width and height of the content area. To calculate the full size of an element, you must also add the padding, borders and margins.

Margin -- auto, margin-top, margin-right,... values

Some possible values for for margin

  • auto - tells the document to automatically put equal left and right margins on our element, centering it on the page.
  • margin-top margin-right margin-bottom margin-left
  • margin: 1px 2px 3px 4px - top, right, bottom, left margin at once (think clock)
 1 element{
 2     margin:auto;
 3     margin: 5px 5px 5px 5px; /* top right bottom left */
 4     margin: 5px; /* set all values at once */
 5 
 6     /* set each margin individualy */
 7     margin-top: 5px;
 8     margin-right: 5px;
 9     margin-bottom: 5px;
10     margin-left: 5px;
11 }

Positioning with the float property!

Floats allow us to control where individual html elements are placed on the page.

1 element{
2     float: /* right, left, none, initial, or inherit */
3 }

Clearing elements with the clear property

Careful with overlapping floats!! --- Large floating elements can often get mixed with non-floating ones, and elements do end up on top of each other.

  • clear:left --- it will move any floating elements on the LHS
  • clear:right -- it will move any floating elements on the RHS
  • clear:both
1 element {
2     clear: /*right, left, or both*/
3 }

Absolute positioning with the position property

Some values for the position property

  • position: static (default) --
  • position:absolute -- positioned relative to the first parent element it has that DOESN'T have position:static value.
1 element {
2     position: /* absolute, static, fixed, relative, initial, inherit*/
3 }
